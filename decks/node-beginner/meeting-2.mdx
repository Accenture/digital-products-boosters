## Setup üõ†

1. [Install Postman](https://www.getpostman.com/)

2. [Get GitHub personal access token](https://help.github.com/en/github/authenticating-to-github/creating-a-personal-access-token-for-the-command-line#creating-a-token)

3. Copy access token and paste into `Authorization` -> `Bearer Token` in Postman

4. To test that it works, run a `GET` request to the following endpoint: https://api.github.com

<p style={{fontSize: '0.55em', fontStyle: 'italic'}}>
Note: Your Access Token should be kept hidden.
</p>

<p style={{fontSize: '0.55em', fontStyle: 'italic'}}>
To learn more about variables on Postman, go here:
https://learning.getpostman.com/docs/postman/variables-and-environments/variables/#environments-in-postman
</p>

# Exercise 5

Using a repository from the solution above, check if that repository has vulnerability alerts enabled

<div style={{ fontSize: '0.6em' }} >

_Note: you need to include `application/vnd.github.dorian-preview+json` in the `Accept` header since this feature is still in preview._

Need help? https://developer.github.com/v3/repos/#enable-vulnerability-alerts

</div>

<MinuteTimer minutes={5} />

---

<LargerLayout>

# Exercise 5 - Answer

<div style={{ fontSize: '0.8em' }} >

- Check if vulnerability alerts are enabled for a repository:
  - GET https://api.github.com/repos/:owner/:repo_name/vulnerability-alerts
  - If Repo is enabled you should see `Status: 204 No Content` in response headers
  - If Repo is NOT enabled you should see `Status: 404 Not Found` in response headers
- Enable vulnerability alerts for a repo:
  - PUT https://api.github.com/repos/:owner/:repo_name/vulnerability-alerts
- Disable vulnerability alerts for a repo:
  - DELETE https://api.github.com/repos/:owner/:repo_name/vulnerability-alerts

</div>

</LargerLayout>
